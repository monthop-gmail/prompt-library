# Open Data Maturity Model บันได Gartner

> openapi opendata คุ้นๆว่า เป็นบันได จาก gartner แต่หาไม่เจอ

โอ้ ผู้ใช้ถามถึง “OpenAPI OpenData” ที่ดูเหมือนจะเป็นขั้นบันไดหรือเฟรมเวิร์กจาก Gartner แต่หาไม่เจอ

น่าจะเป็นการสับสนระหว่างสองแนวคิดคือ **Open API** กับ **Open Data** ซึ่ง Gartner เคยพูดถึงทั้งคู่แต่คนละบริบท

- Open API ส่วนมากจะอ้างอิงมาตรฐาน technical ของ REST API
- Open Data เป็นแนวคิดการเปิดข้อมูลสาธารณะ

ที่ Gartner น่าจะพูดถึงคือ API Strategy Maturity Model หรือ Data and Analytics Maturity Model ซึ่งทั้งสองอย่างมีลักษณะเป็นขั้นบันได

ผู้ใช้น่าจะกำลังนึกถึง **“Open Data Maturity Model”** หรือบางครั้งเรียกว่า **“Data Openness Continuum”** ซึ่ง Gartner หรือองค์กรอื่นๆ ใช้ประเมินระดับการเปิดเผยข้อมูลขององค์กรหรือภาครัฐ

แนวคิดนี้มักถูกนำเสนอเป็น **“บันได” (Ladder) หรือขั้นบันได (Staircase)** แสดงระดับความเปิดของข้อมูลตั้งแต่ปิดสนิทไปจนถึงเปิดให้ใช้อย่างเต็มที่ โดยมักเชื่อมโยงกับ **Open Data** และ **Open API**

---

## รูปแบบบันไดของ Open Data / API ที่พบได้บ่อย

1. **ข้อมูลภายใน (Internal)** – ข้อมูลปิด ใช้ในองค์กรเท่านั้น  
2. **ข้อมูลแบบเลือกเปิด (Selective Sharing)** – เปิดให้บางพันธมิตรใช้ผ่าน API แบบกำหนดสิทธิ  
3. **ข้อมูลสาธารณะ (Public Data)** – เปิดเป็นไฟล์ให้ดาวน์โหลด (เช่น PDF, Excel)  
4. **API สาธารณะ (Public API)** – ให้เข้าถึงข้อมูลแบบเรียลไทม์ผ่าน API  
5. **ข้อมูลแบบเปิดเต็มรูปแบบ (Open Data)** – ข้อมูลเป็นโครงสร้าง ใช้มาตรฐานเปิด (เช่น JSON, XML) มี License เปิดให้ใช้เสรี

## Gartner และ Open Data / API

Gartner กล่าวถึง Open Data และ API Strategy ในหลายบริบท เช่น

- “API Maturity Model”
- “Open Data Governance”
- “Digital Government Maturity Model”

แต่ถ้าคุณนึกถึงรูป “บันได Open Data” ที่เผยแพร่ในไทย อาจมาจาก **สำนักงานรัฐบาลดิจิทัล (DGA)** หรือ **World Bank** ที่ปรับใช้แนวคิดคล้ายๆ กัน

แนะนำคำค้นเพื่อหารูปหรือสไลด์:
- “Open Data Ladder”
- “Data Openness Spectrum”
- “API Maturity Model Gartner”

## บันได 5 ขั้นการเปิดเผยข้อมูลสาธารณะของรัฐ (DGA)

สำนักงานรัฐบาลดิจิทัล (DGA) มี “แนวทางการเปิดเผยข้อมูลสาธารณะของรัฐ” เป็น **5-level maturity model** มักถูกวาดเป็น “บันได” ดังนี้ [web:5][web:8]

### ขั้นที่ 1: ข้อมูลพื้นฐาน (Basic)

- **ลักษณะ:** ไฟล์ทั่วไป เช่น PDF, Word, รูปภาพ
- **จุดเด่น:** ดู/ดาวน์โหลดได้ แต่ machine-readable ไม่ดี
- **ตัวอย่าง:** รายงานประจำปี PDF, ภาพสแกนเอกสาร

### ขั้นที่ 2: ข้อมูลที่มีโครงสร้าง (Structured)

- **ลักษณะ:** ข้อมูลอยู่ในรูปตาราง เช่น Excel, CSV
- **จุดเด่น:** นำไปคำนวณต่อได้ แต่ยังไม่เรียลไทม์
- **ตัวอย่าง:** สถิติในไฟล์ Excel, ข้อมูลรายจ่าย CSV

### ขั้นที่ 3: ข้อมูลแบบเปิด (Open Format)

- **ลักษณะ:** ใช้รูปแบบเปิด เช่น CSV, XML, JSON
- **จุดเด่น:** ใช้งานได้กับซอฟต์แวร์หลากหลาย ไม่มีข้อจำกัดลิขสิทธิ์เชิงพาณิชย์
- **ตัวอย่าง:** ข้อมูล GDP ใน CSV/JSON, ข้อมูลอากาศ XML

### ขั้นที่ 4: ข้อมูลแบบเปิดพร้อม API (Open with API)

- **ลักษณะ:** เปิดผ่าน API
- **จุดเด่น:** ดึงข้อมูลแบบเรียลไทม์ เชื่อมแอปและบริการอื่นได้อัตโนมัติ
- **ตัวอย่าง:** API ตารางรถโดยสาร, API ตำแหน่งงานว่าง

### ขั้นที่ 5: ข้อมูลแบบเปิดที่เชื่อมโยงกัน (Linked Open Data)

- **ลักษณะ:** ใช้ Linked Data เช่น RDF, SPARQL เชื่อมโยงหลายชุดข้อมูล
- **จุดเด่น:** สร้างเครือข่ายข้อมูลข้ามแหล่ง รองรับ Semantic Web
- **ตัวอย่าง:** ข้อมูลงบประมาณเชื่อมกับโครงการและผู้รับเหมา, ข้อมูลวิจัยเชื่อมกับผลงานอื่น

## เป้าหมายของบันได 5 ขั้น DGA

- ให้หน่วยงานรัฐพัฒนาการเปิดข้อมูลอย่างเป็นขั้นตอน  
- เพิ่มมูลค่าและประโยชน์ของข้อมูลสาธารณะ  
- สนับสนุนนวัตกรรมและบริการใหม่จากเอกชนและประชาชน  
- เพิ่มความโปร่งใสและการมีส่วนร่วมของสังคม [web:8]

## ขั้นที่ 5 vs “Open MCP สำหรับ AI”

คำถาม: ถ้าจะทำ “ขั้นที่ 5 เป็น Open MCP สำหรับ AI” จะซ้ำกับขั้นที่ 5 เดิมไหม?

**คำตอบ:** ไม่ซ้ำกัน แต่เป็นการ “ต่อยอด” ขั้นที่ 5 เดิม

- ขั้นที่ 5 เดิม: เน้น “โครงสร้างและความเชื่อมโยงของข้อมูล” (Linked Open Data)
- Open MCP สำหรับ AI: เน้น “โปรโตคอล/วิธีที่ AI เข้าถึงและใช้ข้อมูล”

จึงอาจมองเป็น:

- ขั้นที่ 5: Linked Open Data
- ขั้นที่ 6 (เสนอแนวคิด): **AI-Ready Open Data / Open MCP for AI**

---

## ตารางเปรียบเทียบ ขั้นที่ 5 เดิม vs ขั้นที่ 5 แบบ Open MCP

| มิติ                 | ขั้นที่ 5 เดิม (Linked Open Data)                          | ขั้นที่ 5 แบบ Open MCP สำหรับ AI                                      |
|----------------------|-------------------------------------------------------------|------------------------------------------------------------------------|
| รูปแบบข้อมูล        | RDF, SPARQL, Ontology                                       | Linked Data + API/endpoint ที่รองรับ MCP                              |
| กลุ่มผู้ใช้หลัก     | นักพัฒนา Semantic Web, นักวิจัย                           | AI Agents, Assistants, นักพัฒนาที่สร้างระบบ AI ใช้ข้อมูลรัฐ          |
| การเข้าถึง          | SPARQL endpoint, data dump                                  | ผ่าน MCP Server เป็น gateway ให้ AI                                   |
| ฟังก์ชัน             | Query, เชื่อมข้อมูลข้ามแหล่ง                              | AI ดึงข้อมูลเรียลไทม์, คำนวณ, ตรวจสถานะ, เคารพสิทธิ์/นโยบาย        |
| ตัวอย่างคำถาม/งาน   | “โครงการ A ผูกกับบริษัทอะไรบ้าง?”                         | “ตอนนี้โครงการนี้ใช้งบไปเท่าไรแล้ว? เสี่ยงตรงไหน?” AI ไปดึงข้อมูลเอง |

ข้อสรุป:

- Linked Open Data = ชั้นข้อมูลและความหมาย
- Open MCP สำหรับ AI = ชั้นโปรโตคอลการเข้าถึงโดย AI

## Government Data ที่เป็นรูปธรรม

ตัวอย่างหมวด **Government Data** ที่นำไปสู่ขั้นที่ 6 ได้ [web:8]

1. **เศรษฐกิจและการเงิน**
   - งบประมาณรายจ่าย
   - ข้อมูลจัดซื้อจัดจ้าง
   - ดัชนีราคา, ภาษีในภาพรวม

2. **สังคมและประชากร**
   - จำนวนประชากร
   - สถิติการเกิด-ตาย
   - ข้อมูลผู้รับสวัสดิการ

3. **โครงสร้างพื้นฐาน**
   - ข้อมูลขนส่งสาธารณะ (เส้นทาง, ตารางเวลา)
   - จราจรเรียลไทม์
   - แผนที่, ที่ดิน, สิ่งแวดล้อม

4. **บริการสาธารณะ**
   - โรงเรียน, โรงพยาบาล, คิวรักษา
   - ข้อมูลธุรกิจจดทะเบียน

5. **วิทยาศาสตร์และสิ่งแวดล้อม**
   - พยากรณ์อากาศ
   - น้ำฝน, ภัยพิบัติ
   - ข้อมูลดาวเทียม
